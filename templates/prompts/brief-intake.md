# Brief intake — required structure

Every Drumbeat launch starts from a structured brief. If a user gives
fewer fields, ask for the rest before classifying tier.

## Required fields

| Field | Why it matters |
|---|---|
| **Product / feature name** | Anchors all assets. |
| **One-sentence description** | If you can't say what it is in one sentence, you can't launch it. |
| **Target audience** | Persona + company stage + role. Drives messaging matrix and channel selection. |
| **Problem solved** | What pain does this address? What does the status quo cost the customer? |
| **Differentiation** | Why this, why now, vs. alternatives (named, not generic). |
| **Proof points** | Data, customer logos, beta results, quotes. If missing, asset gen will leave `[PROOF NEEDED]` markers. |
| **Launch date** | Determines urgency, sequencing, asset prioritization. |
| **Strategic context** | Which company priority this launch serves (e.g., "expand into mid-market," "defend against {competitor}"). |
| **Distribution channels** | What teams + comms surfaces are available (sales, CS, blog, email list, social, partners). |
| **Risks / sensitivities** | Regulated industry, competitive response expected, fragile customer relationships, brand consistency concerns. |

## Optional but valuable

| Field | When it helps |
|---|---|
| **Pricing change** | If launch involves new pricing, messaging matrix and FAQ depth change materially. |
| **Existing positioning doc** | If the company has one, anchor to it instead of reinventing. |
| **Prior launch retros** | Avoid repeating mistakes; reuse what worked. |
| **Stakeholder list** | Identifies who must approve copy (PM, eng lead, exec sponsor, sales lead). |

## Brief intake response template (Claude uses this)

When asking for missing fields, format:

> Got it — {what they shared}. To classify the tier and build messaging,
> I need:
>
> 1. {missing field 1} — {why it matters in one phrase}
> 2. {missing field 2} — {why}
> 3. ...
>
> You can paste rough notes; I'll structure them.

Never ask for everything at once if the user clearly only has partial
information. Prioritize: audience > problem > differentiation > proof >
date.
